# 项目规范：Java 面试八股文笔记

## 项目定位

本项目用于系统记录 Java 后端开发面试的"八股文"知识点，覆盖常见面试题与标准答案，便于复习与检索。

## 目录结构

```
技能/
├── Java基础/          # 集合、并发、JVM、泛型、反射等
├── 框架/              # Spring、MyBatis、Spring Boot 等
├── 数据库/            # MySQL、Redis 等
├── 中间件/            # MQ、ZooKeeper、Elasticsearch 等
├── 分布式/            # 分布式锁、事务、微服务等
├── 计算机网络/        # HTTP、TCP/IP 等
├── 操作系统/          # Linux、进程线程等
├── 设计模式/          # 23 种设计模式
├── 算法与数据结构/     # 常见算法题
└── 项目实战/          # 个人项目经验总结
```

## 笔记编写规范

### 文件名

- 使用中文命名，格式：`知识点名称.md`
- 例如：`HashMap源码分析.md`、`Spring事务传播机制.md`

### 内容结构

每个笔记文件建议遵循以下结构：

```markdown
# 标题（知识点名称）

## 概述
1-2 句话概括核心概念。

## 核心原理
分步骤/分要点阐述底层原理。

## 常见面试题

### Q1：具体问题？
**答：** 简明扼要回答，必要时配合代码示例。

### Q2：具体问题？
**答：** ...

## 补充说明
- 易混淆点对比
- 版本差异（如 JDK 7 vs JDK 8）
- 实际应用场景
```

### 内容要求

1. **语言**：中文
2. **准确性优先**：答案必须准确，不确定的内容标注"待确认"
3. **代码示例**：关键代码需可编译运行，标明 JDK 版本
4. **图文结合**：复杂流程（如并发流程、事务流程）可用 ASCII 图或 Mermaid 图说明
5. **长度控制**：单个笔记控制在 2000 字以内，过长则拆分
6. **标注考点**：高频面试题用 `> [!IMPORTANT]` 标记

## 代码规范

- Java 代码使用 ```java 代码块
- 类名、方法名需符合 Java 命名规范
- 注释使用中文，解释关键逻辑

## 协作约定

- 新增笔记时先确认分类，避免重复
- 修改他人笔记需保留核心内容，仅做补充或修正
- 提交信息格式：`分类: 具体操作`，如 `框架: 新增Spring事务传播机制笔记`
