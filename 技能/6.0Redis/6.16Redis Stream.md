### 一、Redis Stream 核心原理
Redis Stream 是 Redis 5.0 引入的持久化、有序、可回溯的消息流结构，专为解决传统 Pub/Sub（无持久化、无 ACK）、List（无消费组、无负载均衡）的缺陷设计，是实现可靠任务队列的最佳 Redis 方案。
#### 1. 核心概念

| 概念 | 通俗解释 |
| --- | --- |
| Stream（流） | 消息的容器，类似「任务清单」，所有任务消息按顺序存入，支持持久化到磁盘。 |
| Message（消息） | 队列中的单个任务，包含：1. 唯一 ID（格式 时间戳-序列号，如 1710888000000-0）；2. 键值对形式的消息体（如 taskId:order_123, data:{...}）。|
| Consumer Group（消费组） | 「消费团队」，多个消费者可加入同一组，实现任务负载均衡（同组消费者不会重复消费同一条消息）。 |
| Consumer（消费者） | 消费组中的「团队成员」，负责处理分配到的任务。 |
| PEL（Pending Entries List） | 消费组的「待确认任务清单」：消息被投递到消费者后，会暂存到 PEL，直到消费者发送 ACK 确认处理完成，才会从 PEL 移除；若消费者崩溃，PEL 可保证任务不丢失。 |
| Last Delivered ID	 | 消费组已投递但未确认的「最后消息 ID」，用于记录消费进度，避免重复投递。 |

### 二、消息队列实现分析
#### 1. 消息发送机制
核心实现：RedisMQTemplate.send(T message) 方法（Stream 重载版本）
```
public <T extends AbstractRedisStreamMessage> RecordId send(T message) {
    try {
        sendMessageBefore(message);
        // 发送消息
        return redisTemplate.opsForStream().add(StreamRecords.newRecord()
                .ofObject(JsonUtils.toJsonString(message)) // 设置内容
                .withStreamKey(message.getStreamKey())); // 设置 stream key
    } finally {
        sendMessageAfter(message);
    }
}

```
##### 工作流程
1. 调用前置拦截器处理
2. 将消息序列化为 JSON 字符串
3. 创建 Stream 记录并发送到指定 Stream
4. 返回消息记录的编号
5. 调用后置拦截器处理
#### 2. 消息消费机制
核心实现：AbstractRedisStreamMessageListener.onMessage() 方法
```
@Override
public void onMessage(ObjectRecord<String, String> message) {
    // 消费消息
    T messageObj = JsonUtils.parseObject(message.getValue(), messageType);
    try {
        consumeMessageBefore(messageObj);
        // 消费消息
        this.onMessage(messageObj);
        // ack 消息消费完成
        redisMQTemplate.getRedisTemplate().opsForStream().acknowledge(group, message);
    } finally {
        consumeMessageAfter(messageObj);
    }
}
```
##### 工作流程：

1. 从 Redis Stream 收到消息后，反序列化为具体的消息对象
2. 调用前置拦截器处理
3. 调用自定义的 onMessage() 方法处理消息
4. 调用 acknowledge() 确认消息消费完成
5. 调用后置拦截器处理

#### 3. 消费者容器配置
```
@Bean(initMethod = "start", destroyMethod = "stop")
@ConditionalOnBean(AbstractRedisStreamMessageListener.class)
public StreamMessageListenerContainer<String, ObjectRecord<String, String>> redisStreamMessageListenerContainer(
        RedisMQTemplate redisMQTemplate, List<AbstractRedisStreamMessageListener<?>> listeners) {
    // 创建容器和配置
    StreamMessageListenerContainer.StreamMessageListenerContainerOptions<String, ObjectRecord<String, String>> containerOptions =
            StreamMessageListenerContainer.StreamMessageListenerContainerOptions.builder()
                    .batchSize(10) // 一次性最多拉取多少条消息
                    .targetType(String.class) // 目标类型
                    .build();
    // 创建 container 对象
    StreamMessageListenerContainer<String, ObjectRecord<String, String>> container =
            StreamMessageListenerContainer.create(redisMQTemplate.getRedisTemplate().getRequiredConnectionFactory(), containerOptions);

    // 注册监听器
    String consumerName = buildConsumerName();
    listeners.parallelStream().forEach(listener -> {
        // 创建 listener 对应的消费者分组
        try {
            redisTemplate.opsForStream().createGroup(listener.getStreamKey(), listener.getGroup());
        } catch (Exception ignore) {
        }
        // 设置 listener 对应的 redisTemplate
        listener.setRedisMQTemplate(redisMQTemplate);
        // 创建 Consumer 对象
        Consumer consumer = Consumer.from(listener.getGroup(), consumerName);
        // 设置 Consumer 消费进度
        StreamOffset<String> streamOffset = StreamOffset.create(listener.getStreamKey(), ReadOffset.lastConsumed());
        // 设置 Consumer 监听
        StreamMessageListenerContainer.StreamReadRequestBuilder<String> builder = StreamMessageListenerContainer.StreamReadRequest
                .builder(streamOffset).consumer(consumer)
                .autoAcknowledge(false) // 不自动 ack
                .cancelOnError(throwable -> false); // 发生异常不取消消费
        container.register(builder.build(), listener);
    });
    return container;
}
```
##### 工作流程：

1. 创建 Stream 消息监听器容器
2. 为每个监听器创建消费者分组
3. 为每个监听器创建消费者实例
4. 设置消费进度为上次消费位置
5. 注册监听器到容器
6. 启动容器
#### 4. 消息重发机制
核心实现：RedisPendingMessageResendJob 类（用于重发未确认的消息）

##### 工作流程：

1. 定时扫描各个 Stream 的未确认消息
2. 将未确认的消息重新投递
3. 确保消息不丢失
#### 5. 消息清理机制

核心实现：RedisStreamMessageCleanupJob 类（用于清理过期的 Stream 消息）

##### 工作流程：

1. 定时清理各个 Stream 的过期消息
2. 避免 Stream 无限增长
3. 优化 Redis 内存使用

### 三、技术特点
1. **集群消费模式**：基于 Redis Stream 的消费者分组机制，实现消息的负载均衡
2. **消息持久化**：Stream 消息持久化到磁盘，支持消息回溯
3. **消息确认机制**：支持消息消费确认（ACK），确保消息不重复消费
4. **消费位置管理**：每个消费者分组维护自己的消费位置
5. **自动配置**：通过 Spring Boot 自动配置，使用简单
6. 消息拦截器：支持消息发送和消费的拦截器
7. **类型安全**：通过泛型确保消息类型的正确性
8. **JSON 序列化**：使用 JSON 格式序列化消息
