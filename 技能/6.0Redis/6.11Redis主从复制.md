# Redis 主从复制原理与实战

## 一、主从复制核心原理
### 1. 主要作用
- **读写分离**：主节点负责写，从节点负责读，分担读压力
- **数据备份**：从节点实时同步主节点数据，提供数据冗余
- **高可用基础**：为哨兵模式、集群模式提供底层支撑

### 2. 核心流程（3个阶段）
#### （1）建立连接阶段
1. 从节点执行 `replicaof` 命令，向主节点发起连接
2. 主从节点完成 **PING/PONG 心跳验证**
3. 若主节点设置了密码，从节点需通过身份验证

#### （2）数据同步阶段
分为 **全量同步** 和 **增量同步** 两种模式：
- **全量同步**（初次连接或断线后无法增量时触发）：
  1. 主节点执行 `BGSAVE` 生成 RDB 快照文件
  2. 主节点将 RDB 文件发送给从节点
  3. 从节点清空旧数据，加载 RDB 文件
  4. 主节点将同步期间的写命令存入 **复制积压缓冲区**，发送给从节点执行
- **增量同步**（PSYNC 2.0 机制，断线重连时优先触发）：
  1. 从节点向主节点发送 `PSYNC` 命令，携带 **复制偏移量** 和 **主节点运行ID**
  2. 若偏移量在主节点的复制积压缓冲区范围内，主节点只发送缺失的写命令
  3. 从节点执行命令，完成同步

#### （3）命令传播阶段
- 主节点执行写命令后，异步将命令发送给所有从节点
- 从节点执行命令，保证数据一致性

### 3. 关键机制
- **复制偏移量**：主从节点各自维护一个偏移量，用于判断数据是否一致
- **复制积压缓冲区**：主节点的一个固定长度队列，存储最近的写命令，用于增量同步
- **心跳检测**：从节点每秒向主节点发送 `REPLCONF ACK <offset>`，主节点据此判断从节点状态

## 二、主从复制实战（一主两从架构）
### 1. 环境准备
- 3 台服务器（或本地 3 个 Redis 实例，端口分别为 6379、6380、6381）
- Redis 版本建议 5.0+（支持 PSYNC 2.0）

### 2. 配置主节点（端口 6379）
修改 `redis.conf` 核心配置：
```conf
# 绑定IP（若需远程访问，改为0.0.0.0或具体IP）
bind 127.0.0.1
# 端口
port 6379
# 开启持久化（可选，建议开启）
save 900 1
save 300 10
save 60 10000
# 复制积压缓冲区大小（默认1MB，根据写压力调整）
repl-backlog-size 64mb
```
启动主节点：
```bash
redis-server redis.conf
```

### 3. 配置从节点（端口 6380、6381）
以 6380 为例，修改 `redis_6380.conf`：
```conf
bind 127.0.0.1
port 6380
# 配置从节点，指向主节点IP和端口
replicaof 127.0.0.1 6379
# 从节点只读（默认开启，防止误写）
replica-read-only yes
```
启动从节点：
```bash
redis-server redis_6380.conf
redis-server redis_6381.conf
```

### 4. 验证主从复制
#### （1）查看复制状态
连接主节点，执行：
```bash
redis-cli -p 6379
127.0.0.1:6379> info replication
```
输出应包含：
```
# Replication
role:master
connected_slaves:2
slave0:ip=127.0.0.1,port=6380,state=online,offset=1234,lag=1
slave1:ip=127.0.0.1,port=6381,state=online,offset=1234,lag=1
```

连接从节点，执行：
```bash
redis-cli -p 6380
127.0.0.1:6380> info replication
```
输出应包含：
```
# Replication
role:slave
master_host:127.0.0.1
master_port:6379
master_link_status:up
```

#### （2）测试数据同步
在主节点写入数据：
```bash
127.0.0.1:6379> set test_key "hello redis replication"
OK
```
在从节点读取数据：
```bash
127.0.0.1:6380> get test_key
"hello redis replication"
```

### 5. 常见问题排查
- **从节点一直处于 `sync` 状态**：检查主节点网络带宽、RDB 生成速度，或增大 `repl-timeout` 参数
- **复制中断**：查看主从节点日志，检查复制偏移量是否超出积压缓冲区范围，可适当增大 `repl-backlog-size`
- **数据不一致**：检查从节点是否开启了 `replica-read-only no`，避免从节点写入数据
