#### 一、Java IO 整体体系

Java IO 主要用于处理设备间的数据传输（如文件、网络、内存），核心分为两大体系：

*   **BIO（Blocking IO，阻塞式 IO）**：传统 IO，本文核心讲解的基础 IO；
*   **NIO（Non-Blocking IO，非阻塞 IO）**：JDK1.4 引入，面向缓冲区、非阻塞，面试常对比 BIO 提问。

| 特性   | BIO（传统 IO）          | NIO（JDK1.4+）             |
| ---- | ------------------- | ------------------------ |
| 模型   | 阻塞式 IO              | 非阻塞式 IO                  |
| 核心对象 | 流（Stream）           | 通道（Channel）+ 缓冲区（Buffer） |
| 处理方式 | 面向流（一次读 1 个字节 / 字符） | 面向缓冲区（批量读写，效率高）          |
| 多路复用 | 不支持                 | 支持（Selector，一个线程处理多个通道）  |
| 适用场景 | 连接数少、操作简单的场景        | 连接数多、高并发场景（如 Netty 框架）   |

#### 二、IO 流的核心分类

1.  按**数据单位**：字节流（8 位）、字符流（16 位，针对文本）；
2.  按**流向**：输入流（读数据）、输出流（写数据）；
3.  按**角色**：节点流（直接操作数据源）、处理流（包装节点流，增强功能，如缓冲）。
