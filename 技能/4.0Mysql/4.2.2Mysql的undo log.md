### undo log（回滚日志）- 保证「原子性」+ 事务回滚 + MVCC

**核心定义**
undo log 是 InnoDB 自带的「逻辑日志」，记录的是：SQL 的「反向操作」（比如你执行update stock set num=99 where id=1，undo log 就记录update stock set num=100 where id=1；你执行delete from order where id=1，undo log 就记录insert into order ...）。
undo log 的核心作用：实现事务回滚、保证原子性，同时是 MVCC 的底层支撑。
**undo log 的工作机制**
undo log 在事务执行的那一刻就开始写入，和 redo log 是成对出现的，核心解决「原子性」的核心诉求：失败就回滚。

1.  事务执行每条修改 SQL（insert/update/delete）→ InnoDB 自动生成对应的 undo log，写入磁盘；
2.  事务执行过程中，如果出现「SQL 报错、业务异常、手动 rollback」→ InnoDB 会读取 undo log，执行「反向操作」，把数据恢复到事务执行前的状态；
3.  事务执行成功 commit 后 → undo log 不会立刻删除，会保留一段时间，用于「MVCC 的读操作」和「崩溃恢复」，后续由后台线程清理。
