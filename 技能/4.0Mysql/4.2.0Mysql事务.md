### 1. 基础核心

*   什么是 MySQL 事务

> 把「一组 SQL 操作」封装成一个「不可分割的执行单元」，这组 SQL 要么 全部执行成功，要么 全部执行失败回滚，不存在「部分成功、部分失败」的中间状态。

*   事务的四大特性：ACID

> 事务四大特性：**原子性（Atomicity）、一致性（Consistency）、隔离性（Isolation）、持久性（Durability）**
>
> 1.  原子性：核心是「不可分割」，事务里的所有 SQL，要么全执行成功，要么全回滚，没有中间态；
> 2.  一致性：核心是「数据完整」，事务执行前和执行后，数据库的业务数据逻辑必须是正确的（比如你下单，库存减少的数量 = 订单商品数量，总金额不变）；
> 3.  隔离性：核心是「互不干扰」，多个事务并发执行时，彼此之间不能互相影响，每个事务都感觉不到其他事务的存在；
> 4.  持久性：核心是「永不丢失」，事务一旦执行成功（commit），修改的数据会永久保存到数据库磁盘，就算 MySQL 崩溃、服务器断电，数据也不会丢失

*   ACID 之间的核心关系：原子性 + 隔离性 + 持久性 → 共同保障 一致性；一致性是事务的「最终目标」，其他三个特性都是实现目标的「手段」。

### 2. MySQL 事务的「底层核心支撑」

*   InnoDB 实现事务的 ACID 四大特性，靠的是 3 个核心底层机制，三者缺一不可，分工明确：

> 原子性 + 持久性 → 由 redo log（重做日志） + undo log（回滚日志） 共同保证；
> 隔离性 → 由 锁机制 + MVCC（多版本并发控制） 共同保证；
> 一致性 → 由上面 4 个机制一起保障（原子 + 隔离 + 持久达标，一致性自然达标）。
