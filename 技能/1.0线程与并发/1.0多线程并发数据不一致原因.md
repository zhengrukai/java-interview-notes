### 1. 原子性问题

> 原子性是指一个操作是不可中断的，要么全部执行成功，要么全部执行失败。在多线程环境下，某些操作虽然看起来是单个操作，但实际上可能由多个步骤组成，不是原子操作。

    // 非原子操作
    count++; // 实际上包含三个步骤：读取count值、加1、写回count

    # 当两个线程同时执行count++时，可能会发生以下情况：

    # 线程A读取count值为10
    # 线程B读取count值为10
    # 线程A执行加1，得到11
    # 线程B执行加1，得到11
    # 线程A写回count值为11
    # 线程B写回count值为11
    # 最终count值为11，而不是期望的12。

### 2. 可见性问题

> 可见性是指当一个线程修改了共享变量的值后，其他线程能够立即看到这个修改。在多线程环境下，由于CPU缓存的存在，可能会导致可见性问题。

**原因：**

*   每个线程都有自己的工作内存（CPU缓存）
*   线程操作共享变量时，会先将变量从主内存复制到工作内存
*   修改后，线程会将修改后的值写回主内存
*   但这个写回操作可能不是立即执行的，导致其他线程读取到的还是旧值

### 3. 有序性问题

> 有序性是指程序执行的顺序按照代码的先后顺序执行。在多线程环境下，由于编译器优化、CPU指令重排序等原因，可能会导致有序性问题

    // 线程1
    a = 1; // 操作1
    flag = true; // 操作2

    // 线程2
    while (!flag) {
        // 等待
    }
    int b = a; // 期望b=1
    # 由于指令重排序，操作1和操作2的执行顺序可能被交换，导致线程2读取到a的值为0（初始值）
