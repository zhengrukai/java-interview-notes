# 1. 继承 Thread 类，重写run()

    // 案例：继承Thread创建线程
    class MyThread extends Thread {
        @Override
        public void run() { // 线程执行逻辑
            for (int i = 0; i < 5; i++) {
                System.out.println(Thread.currentThread().getName() + "：" + i);
                try {
                    Thread.sleep(100); // 休眠100ms，模拟业务
                } catch (InterruptedException e) {
                    e.printStackTrace();
                }
            }
        }
    }

    public class ThreadCreateDemo1 {
        public static void main(String[] args) {
            // 创建线程实例
            MyThread t1 = new MyThread();
            MyThread t2 = new MyThread();
            // 设置线程名（可选）
            t1.setName("线程1");
            t2.setName("线程2");
            // 启动线程（必须调用start()，而非直接run()）
            t1.start();
            t2.start();
        }
    }

*   start()：真正启动线程，JVM 会调用run()方法；直接调用run()只是普通方法调用，不会创建新线程；
*   缺点：Java 单继承限制，继承 Thread 后无法继承其他类。

# 2. 实现 Runnable 接口（推荐）

    // 案例：实现Runnable创建线程
    class MyRunnable implements Runnable {
        @Override
        public void run() {
            for (int i = 0; i < 5; i++) {
                System.out.println(Thread.currentThread().getName() + "：" + i);
                try {
                    Thread.sleep(100);
                } catch (InterruptedException e) {
                    e.printStackTrace();
                }
            }
        }
    }

    public class ThreadCreateDemo2 {
        public static void main(String[] args) {
            // 创建Runnable实例
            MyRunnable runnable = new MyRunnable();
            // 包装为Thread对象
            Thread t1 = new Thread(runnable, "线程1");
            Thread t2 = new Thread(runnable, "线程2");
            // 启动
            t1.start();
            t2.start();
        }
    }

*   规避单继承限制，推荐实际开发使用；
*   多个线程可共享同一个Runnable实例的资源（比如共享变量）。

# 3. 实现 Callable+FutureTask（有返回值 / 可抛异常）

    import java.util.concurrent.Callable;
    import java.util.concurrent.FutureTask;

    // 案例：有返回值的线程创建
    class MyCallable implements Callable<Integer> {
        @Override
        public Integer call() throws Exception { // 可抛异常，有返回值
            int sum = 0;
            for (int i = 1; i <= 100; i++) {
                sum += i;
                Thread.sleep(10);
            }
            return sum; // 返回1-100的和
        }
    }

    public class ThreadCreateDemo3 {
        public static void main(String[] args) throws Exception {
            // 创建Callable实例
            Callable<Integer> callable = new MyCallable();
            // 包装为FutureTask（实现了Runnable）
            FutureTask<Integer> futureTask = new FutureTask<>(callable);
            // 启动线程
            Thread t = new Thread(futureTask, "计算线程");
            t.start();
            // 获取返回值（会阻塞，直到线程执行完成）
            Integer result = futureTask.get();
            System.out.println(t.getName() + "计算结果：" + result); // 输出5050
        }
    }

*   解决Runnable无返回值、无法抛受检异常的问题；
*   futureTask.get()会阻塞主线程，直到子线程返回结果。

# 4. 线程池方式（生产推荐）

```java
import java.util.concurrent.*;

public class ThreadCreateDemo4 {
    public static void main(String[] args) {
        // 生产环境推荐：手动创建 ThreadPoolExecutor
        ExecutorService pool = new ThreadPoolExecutor(
            2, 4, 60L, TimeUnit.SECONDS,
            new ArrayBlockingQueue<>(100),
            Executors.defaultThreadFactory(),
            new ThreadPoolExecutor.CallerRunsPolicy()
        );

        // 提交 Runnable 任务
        pool.execute(() -> System.out.println(Thread.currentThread().getName() + "：执行任务"));

        // 提交 Callable 任务（有返回值）
        Future<String> future = pool.submit(() -> {
            return Thread.currentThread().getName() + "：返回结果";
        });

        try {
            System.out.println(future.get()); // 获取返回值
        } catch (Exception e) {
            e.printStackTrace();
        }

        pool.shutdown();
    }
}
```

*   线程复用，避免频繁创建/销毁线程的开销；
*   可控制最大并发数，防止资源耗尽；
*   阿里规范要求：**禁止使用 Executors 创建**，必须通过 ThreadPoolExecutor 手动指定参数。

### 四种创建方式对比

| 方式 | 返回值 | 异常处理 | 线程复用 | 适用场景 |
|------|--------|---------|---------|---------|
| 继承 Thread | 无 | 仅 RuntimeException | 否 | 简单场景，不推荐 |
| 实现 Runnable | 无 | 仅 RuntimeException | 否 | 无返回值任务 |
| Callable + FutureTask | 有 | 可抛受检异常 | 否 | 需要返回值/异常 |
| 线程池 | 可有可无 | 灵活处理 | **是** | **生产推荐** |

# 线程生命周期（6 种状态）

### 完整状态转换图（面试必画）

```
                    ┌──────────────────────────────────────────┐
                    │              RUNNABLE                     │
    new Thread()    │  ┌─────────┐          ┌─────────┐       │
  ───────────────>  │  │  Ready  │ ◄──────► │ Running │       │
       NEW          │  │ (就绪)   │  CPU调度   │ (运行中) │       │
                    │  └─────────┘          └────┬────┘       │
                    └───────┬──────────────────┬─┼────────────┘
                            │                  │ │      run()结束
                            │                  │ │     ──────────►  TERMINATED
                    ┌───────┴───────┐   ┌──────┴─┴──────┐
                    │   BLOCKED     │   │   WAITING /    │
                    │ (等synchronized│   │ TIMED_WAITING  │
                    │   锁释放)      │   │ (wait/join/    │
                    └───────────────┘   │  sleep/park)   │
                                        └────────────────┘
```

**Java 6 种状态与触发条件**：

| 状态 | 触发方式 | 回到 RUNNABLE |
|------|---------|--------------|
| **NEW** | `new Thread()` 未调用 start() | `start()` |
| **RUNNABLE** | 包含 OS 的 Ready + Running | — |
| **BLOCKED** | 等待 synchronized 锁 | 获取到锁 |
| **WAITING** | `Object.wait()` / `Thread.join()` / `LockSupport.park()` | `notify()` / 线程结束 / `unpark()` |
| **TIMED_WAITING** | `sleep(ms)` / `wait(ms)` / `join(ms)` / `parkNanos()` | 超时 或 被唤醒 |
| **TERMINATED** | run() 执行完毕或抛出异常 | 不可逆 |

**注意**：`BLOCKED` 只针对 synchronized；Lock.lock() 阻塞的线程状态是 `WAITING`（内部用 LockSupport.park()）

# 线程生命周期控制方法

## 1. sleep(long millis)：线程休眠（不释放锁）

> Thread.sleep(1000); // 休眠1秒

## 2. join()/join(long millis)：等待线程终止

> t1.join(); // 无参：等待t1完全结束
> t1.join(2000); // 有参：最多等待2秒，超时后主线程继续

## 3. yield()：线程让步

> Thread.yield(); // 让步

*   yield()是提示性的，JVM 不一定会采纳（比如只有一个线程时，让步后会立即重新执行）；
*   让步后线程仍处于可运行状态，而非阻塞状态。

# 一、Runnable 接口：无返回值的任务抽象（基础）

1.  **核心定义（源码级）**
    Runnable是一个**函数式接口**（JDK 8 后标注@FunctionalInterface），仅定义一个无返回值、不抛受检异常的run()方法：

    // java.lang.Runnable 核心源码
    @FunctionalInterface
    public interface Runnable {
    /\*\*
    \* 线程要执行的任务逻辑，无返回值，仅能抛出运行时异常（RuntimeException）
    \*/
    void run();
    }

2.  **底层原理**
    Runnable是 Java 线程模型的核心抽象，其设计原理是将 “线程” 和 “任务” 解耦：

*   Thread类负责线程的生命周期管理（启动、暂停、终止）；
*   Runnable负责定义线程要执行的具体业务逻辑；
*   Thread类的构造器接收Runnable实例作为 “任务载体”，当调用Thread.start()后，JVM 会在新线程中执行Runnable.run()方法。

1.  **Runnable 的核心限制（设计缺陷）**

*   无返回值：任务执行完成后，无法向调用方返回结果；
*   异常限制：run()不能抛出受检异常（如IOException），必须在内部捕获，否则编译报错；
*   无法取消：任务一旦启动，没有原生机制取消执行（需手动实现标志位）。

# 二、Callable 接口：有返回值的任务抽象（JUC 补充）

1.  **核心定义（源码级）**
    Callable同样是函数式接口（JUC 包下），定义了一个有泛型返回值、可抛任意异常的call()方法：

    // java.util.concurrent.Callable 核心源码
    @FunctionalInterface
    public interface Callable\<V> {
    /\*\*
    \* 线程要执行的任务逻辑，有返回值（泛型V），可抛出任意异常（包括受检异常）
    \* @return 任务执行结果
    \* @throws Exception 任意异常
    \*/
    V call() throws Exception;
    }

*   泛型V：指定任务返回值的类型；
*   call()：任务执行体，可抛Exception（包括受检异常），解决了Runnable的异常限制。

1.  **底层原理**
    Callable不能直接被Thread接收（因为Thread仅支持Runnable），其核心执行原理依赖桥接类FutureTask：

**核心桥接逻辑：FutureTask（实现 RunnableFuture 接口）**
RunnableFuture继承了Runnable和Future，是连接Callable和Thread的关键：

    // 简化的RunnableFuture接口定义
    public interface RunnableFuture<V> extends Runnable, Future<V> {
        void run();
    }

    // FutureTask实现了RunnableFuture，核心作用：
    // 1. 包装Callable任务；
    // 2. 实现Runnable，让Thread可以执行；
    // 3. 实现Future，提供获取返回值、取消任务的能力。

**FutureTask 的核心执行原理**
1、FutureTask构造时接收Callable实例，将其封装为内部任务；
2、当Thread执行FutureTask.run()时，内部会调用Callable.call()；
3、call()的执行结果 / 异常会被FutureTask缓存；
4、调用FutureTask.get()时，会阻塞等待call()执行完成，并返回结果（或抛出异常）。
执行链路不同：

*   Runnable：Thread.start() → Thread.run() → Runnable.run()；
*   Callable：Thread.start() → Thread.run() → FutureTask.run() → Callable.call() → 结果缓存到FutureTask → 调用get()获取结果。
