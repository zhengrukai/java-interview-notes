在 Java 中，**强引用（Strong Reference）** 和 **弱引用（Weak Reference）** 是对象引用的两种类型，核心区别在于它们对垃圾回收（GC）行为的影响 —— 这也是 Java 为了灵活控制对象生命周期、避免内存泄漏设计的机制。

### 强引用
#### 1. 定义
这是 Java 中最常见、默认的引用类型。我们日常写代码时创建的对象引用，几乎都是强引用。
#### 2. 核心特点
- 只要强引用存在，被引用的对象就会一直存活在内存中；
- 即使 JVM 内存不足抛出 OutOfMemoryError，GC 也不会回收强引用指向的对象。
#### 3. 代码示例
```
public class StrongReferenceDemo {
    public static void main(String[] args) {
        // 创建强引用：obj 就是指向 Object 对象的强引用
        Object obj = new Object();
        
        // 只有当强引用被显式置为 null，或超出作用域（比如方法执行完），对象才失去强引用
        obj = null; // 此时 Object 对象才可能被 GC 回收
    }
}
```
#### 4. 常见场景
日常开发中所有直接赋值的引用都是强引用，比如：
```
String str = "hello"; // str 是强引用
List<String> list = new ArrayList<>(); // list 是强引用
```
### 弱引用
#### 1. 定义
弱引用是一种 “弱关联” 的引用，需要通过 java.lang.ref.WeakReference 类显式创建，无法直接通过赋值生成。
#### 2. 核心特点
- 弱引用本身不影响对象的回收：只要对象没有任何强引用，仅被弱引用指向，GC 一旦扫描到就会回收该对象；
- 弱引用可以配合 ReferenceQueue 使用，当被引用的对象被回收后，弱引用本身会被加入到队列中，方便后续清理。
#### 3. 代码示例
```
import java.lang.ref.WeakReference;

public class WeakReferenceDemo {
    public static void main(String[] args) {
        // 1. 创建一个普通对象，并用强引用指向它
        Object target = new Object();
        // 2. 创建弱引用，指向这个对象
        WeakReference<Object> weakRef = new WeakReference<>(target);
        
        // 此时：target 是强引用，weakRef 是弱引用 → 对象不会被 GC 回收
        System.out.println("回收前，弱引用获取对象：" + weakRef.get()); // 输出：java.lang.Object@xxx
        
        // 3. 断开强引用（关键：让对象仅被弱引用指向）
        target = null;
        
        // 4. 手动触发 GC（注意：生产环境不建议手动调用，仅用于演示）
        System.gc();
        // 给 GC 一点执行时间
        try {
            Thread.sleep(100);
        } catch (InterruptedException e) {
            e.printStackTrace();
        }
        
        // 5. 此时对象已被 GC 回收，弱引用获取返回 null
        System.out.println("回收后，弱引用获取对象：" + weakRef.get()); // 输出：null
    }
}
```
#### 4. 常见场景
弱引用主要用于缓存场景（避免缓存对象长期占用内存导致泄漏），比如：
- Java 中的 WeakHashMap：键（key）是弱引用类型，当键没有强引用时，GC 会回收对应的键值对，常用于缓存、临时数据存储；
- ThreadLocal 的内部实现：依赖弱引用避免线程退出后 ThreadLocalMap 内存泄漏。
