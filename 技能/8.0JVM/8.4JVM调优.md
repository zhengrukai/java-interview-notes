#### 1. 调优目标

*   吞吐量优先：用户代码运行时间 / 总时间占比高（如批处理）；
*   延迟优先：STW 时间短、接口响应快（如 Web 应用）。

#### 2. 高频 JVM 参数

    # 堆内存（核心，建议 Xms=Xmx 避免频繁扩容）
    -Xms2g  # 初始堆内存
    -Xmx2g  # 最大堆内存
    -Xmn1g  # 新生代内存（建议堆的 1/3~1/2）
    -XX:SurvivorRatio=8  # Eden:S0:S1=8:1:1（默认）

    # 元空间
    -XX:MetaspaceSize=256m  # 元空间初始大小
    -XX:MaxMetaspaceSize=512m  # 元空间最大大小

    # 收集器
    -XX:+UseParallelGC  # JDK8 默认（吞吐量优先）
    -XX:+UseG1GC  # G1 收集器（低延迟）

    # GC 日志（调优必备）
    -XX:+PrintGCDetails  # 打印详细 GC 日志
    -XX:+PrintGCTimeStamps  # 打印 GC 时间戳
    -Xloggc:/tmp/gc.log  # 日志输出到文件

#### 3. 调优步骤

1.  监控：用 jstat、jvisualvm、jconsole 监控 GC 频率、STW 时间、内存使用；
2.  定位问题：如 Full GC 频繁、STW 过长、OOM；
3.  调整参数：优化堆内存、新生代比例、收集器等；
4.  验证：观察指标是否达标，反复迭代。
