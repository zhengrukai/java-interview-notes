#### 1. 核心概念

分布式缓存是将缓存数据存储在独立的缓存服务（如 Redis）中，供分布式系统所有节点共享访问。解决了单机缓存（如 HashMap）数据不共享、一致性差的问题，核心价值是减轻数据库压力、提升系统响应速度。

#### 2. 核心要求

*   数据一致性：缓存与数据库数据尽可能一致；
*   高可用：缓存服务无单点故障；
*   高性能：基于内存读写，响应快；
*   可选持久化：防止缓存重启后数据丢失。

#### 3. Java 实现（Redis + Spring Cache，简化开发）

Spring Cache 提供注解式缓存操作，结合 Redis 实现分布式缓存，无需手动编写缓存逻辑。
**前置条件：**

    <!-- Maven 依赖 -->
    <dependency>
        <groupId>org.springframework.boot</groupId>
        <artifactId>spring-boot-starter-cache</artifactId>
    </dependency>
    <dependency>
        <groupId>org.springframework.boot</groupId>
        <artifactId>spring-boot-starter-data-redis</artifactId>
    </dependency>

<!---->

    # application.yml 配置
    spring:
      redis:
        host: 127.0.0.1
        port: 6379
        password: your_redis_pwd
      cache:
        type: redis
        redis:
          time-to-live: 3600000 # 缓存默认过期1小时
          key-prefix: "app:cache:" # 缓存key前缀
          cache-null-values: false # 不缓存null值（避免缓存穿透）

**核心代码：**

    import org.springframework.cache.annotation.CacheEvict;
    import org.springframework.cache.annotation.CachePut;
    import org.springframework.cache.annotation.Cacheable;
    import org.springframework.stereotype.Service;

    @Service
    // 统一指定缓存名称
    @CacheConfig(cacheNames = "userCache")
    public class UserCacheService {

        /**
         * 查询缓存：命中则返回缓存，未命中则查库并缓存
         */
        @Cacheable(key = "#userId", unless = "#result == null")
        public User getUserById(Long userId) {
            // 实际项目中替换为数据库查询（MyBatis/DAO）
            System.out.println("从数据库查询用户：" + userId);
            return new User(userId, "张三", 25);
        }

        /**
         * 更新缓存：更新数据库后同步更新缓存
         */
        @CachePut(key = "#user.id")
        public User updateUser(User user) {
            System.out.println("更新数据库用户：" + user);
            return user;
        }

        /**
         * 删除缓存：删除数据库后删除对应缓存
         */
        @CacheEvict(key = "#userId")
        public void deleteUser(Long userId) {
            System.out.println("删除数据库用户：" + userId);
        }

        // 模拟用户实体
        public static class User {
            private Long id;
            private String name;
            private Integer age;

            public User(Long id, String name, Integer age) {
                this.id = id;
                this.name = name;
                this.age = age;
            }
            // getter/setter 省略
        }
    }

**关键解释：**

*   @Cacheable：查询缓存注解，优先读缓存，无则执行方法并缓存；
*   @CachePut：更新缓存注解，执行方法后更新缓存；
*   @CacheEvict：删除缓存注解，删除指定 key 或全部缓存；
*   核心问题处理：
    *   缓存穿透：请求不存在的 key 导致查库 → 解决方案：缓存 null 值（短过期）、布隆过滤器；
    *   缓存击穿：热点 key 过期瞬间大量请求打向数据库 → 解决方案：热点 key 永不过期、分布式锁；
    *   缓存雪崩：大量 key 同时过期 → 解决方案：过期时间加随机值、Redis 集群。
