MyBatis 是一款轻量级的持久层框架，核心目标是简化 JDBC 操作，将 SQL 与 Java 代码解耦，同时保留 SQL 编写的灵活性。下面从「核心组件」和「完整工作流程」两个维度，由浅入深讲解其底层原理。

## 一、核心组件

| 组件 | 核心作用 |
| --- | --- |
| SqlSessionFactory | 「工厂类」，负责创建 SqlSession，全局唯一（单例），由配置信息构建。 |
| SqlSession | 「会话对象」，应用与数据库的一次会话，非线程安全，用于执行 SQL、获取 Mapper。 |
| Mapper 接口 | 「动态代理」，MyBatis 为 Mapper 接口生成代理对象，接口方法映射对应 SQL。 |
| Executor | 「执行器」，SqlSession 的底层执行器，负责执行 SQL（有基本 / 重用 / 批量三种）。 |
| StatementHandler | 处理 JDBC 的 Statement（PreparedStatement），封装参数设置、SQL 执行。 |
| ParameterHandler | 处理 SQL 参数（将 Java 对象转换为 JDBC 能识别的参数类型）。 |
| ResultSetHandler | 处理查询结果（将 JDBC 的 ResultSet 转换为 Java 实体对象）。 |
| Configuration | 「全局配置容器」，存储 MyBatis 所有配置信息（数据源、映射规则、插件等）。 |

## 二、完整工作流程（从启动到执行 SQL）

```
flowchart TD
    A[加载配置文件] --> B[构建SqlSessionFactory]
    B --> C[创建SqlSession]
    C --> D[获取Mapper代理对象]
    D --> E[匹配SQL语句]
    E --> F[参数处理+执行SQL]
    F --> G[结果集转换]
    G --> H[返回结果+关闭Session]
```
1. **加载配置文件**

MyBatis 启动时，加载核心配置文件（mybatis-config.xml）和 Mapper 映射文件（XxxMapper.xml），或解析注解（如 @Select），将配置信息（数据源、事务、SQL 映射规则等）解析并存储到 Configuration 对象中。

2. **构建 SqlSessionFactory**

通过 SqlSessionFactoryBuilder 读取 Configuration，创建 SqlSessionFactory（通常用单例模式，因为创建成本高）。

3. **创建 SqlSession**

调用 SqlSessionFactory.openSession() 创建 SqlSession，默认生成 DefaultSqlSession，关联 Executor 和 Configuration（可指定事务是否自动提交）。

4. **获取 Mapper 代理对象**

调用 sqlSession.getMapper(XxxMapper.class)，MyBatis 基于 JDK 动态代理为 Mapper 接口生成代理对象（而非手动实现接口）。

5. **匹配 SQL 语句**

调用 Mapper 接口方法时，代理对象会根据「接口全类名 + 方法名」匹配 Mapper 映射中的 SQL ID（如 com.user.mapper.UserMapper.selectById）。

6. **参数处理 + 执行 SQL**

- ParameterHandler 将 Java 参数（如 userId=1）设置到 SQL 中，生成可执行的 PreparedStatement；
- Executor 调用 StatementHandler 执行 SQL，与数据库交互。

7. **结果集转换**

ResultSetHandler 将 JDBC 返回的 ResultSet 结果集，按映射规则（如 resultMap）转换为 Java 实体对象（如 User）。

8. **返回结果 + 关闭 Session**

将转换后的结果返回给调用方，执行完毕后关闭 SqlSession（释放数据库连接，避免资源泄漏）。

## 总结
1. MyBatis 核心是**解耦**：将 SQL 从 Java 代码中抽离，通过配置 / 注解映射，同时用动态代理简化 Mapper 接口的实现；
2. 工作流程核心是「工厂模式 + 动态代理」：SqlSessionFactory 生产 SqlSession，Mapper 接口通过动态代理关联 SQL，底层封装 JDBC 操作；
3. 关键组件分工明确：Executor 负责执行，StatementHandler/ParameterHandler/ResultSetHandler 封装 JDBC 细节，Configuration 存储所有配置。
