### 核心区别

*   `==` 是运算符：既可以比基本类型（比值），也可以比引用类型（比地址）。
*   `equals()` 是方法：只能用于引用类型，默认比地址，但重写后可自定义比内容。

> String, Integer, List 重写后，只比内容

#### String 类的特殊情况（常量池）

    String s1 = "abc"; // 直接在“字符串常量池”中创建
    String s2 = "abc"; // 复用常量池中的对象（地址相同）
    String s3 = new String("abc"); // 强制在堆中新建对象（地址不同）

    System.out.println(s1 == s2); // true（常量池复用）
    System.out.println(s1 == s3); // false（堆 vs 常量池）
    System.out.println(s1.equals(s3)); // true（内容相同）

#### Integer 类的缓存机制（-128\~127）

    Integer i1 = 127; // 缓存范围内，复用对象
    Integer i2 = 127;
    Integer i3 = 128; // 超出缓存，新建对象
    Integer i4 = 128;

    System.out.println(i1 == i2); // true（缓存复用）
    System.out.println(i3 == i4); // false（新建对象）
    System.out.println(i3.equals(i4)); // true（内容相同）

#### 重写 equals() 的 5 个原则

1.  **自反性**：x.equals(x) 必须为 true；
2.  **对称性**：x.equals(y) 与 y.equals(x) 结果一致；
3.  **传递性**：若 x=y 且 y=z，则 x=z；
4.  **一致性**：多次调用结果不变；
5.  **非空性**：x.equals(null) 必须为 false。
