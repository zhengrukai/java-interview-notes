Map 存储 key-value 键值对，key 不可重复，value 可重复。
#### 1. HashMap（核心实现）
- **底层结构**：JDK 1.7 为「数组 + 链表」，JDK 1.8 为「数组 + 链表 + 红黑树」（链表长度≥8 且数组长度≥64 时转为红黑树，避免链表过长导致查询退化）。
- **核心参数**：默认初始容量 16，负载因子 0.75（当元素数量≥容量 × 负载因子时，扩容为原来的 2 倍）。
- **put 流程**：计算 key 的 hash 值 → 定位数组索引 → 若索引位置为空则直接插入，否则遍历链表 / 红黑树插入（若 key 已存在则覆盖 value） → 插入后判断是否需要扩容。
- **特点**：非线程安全，允许 null 键和 null 值；查询 / 插入 / 删除效率高（O (1) 或 O (logn)）。
#### 2. 其他 Map 实现

| 实现类 | 底层结构 | 核心特点 | 适用场景 |
| --- | --- | --- | --- |
| LinkedHashMap | 继承 HashMap + 双向链表 | 维护 key 的插入顺序或访问顺序；性能略低于 HashMap。 | 需要保证顺序的键值对存储 |
| TreeMap | 红黑树 | 按 key 自然排序或自定义比较器排序；查询 / 插入 / 删除为 O (logn) | 排序后的键值对存储 |
| Hashtable | 数组 + 链表 | 线程安全（方法加 synchronized），效率低；不允许 null 键和 null 值。 | 线程安全要求高的旧系统 |
| ConcurrentHashMap | 数组 + 链表 / 红黑树 | 线程安全（JDK 1.7 分段锁，JDK 1.8 CAS + synchronized）；效率远高于 Hashtable。 | 高并发场景下的键值对存储 |
