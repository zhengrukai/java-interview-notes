# Spring Boot Security 原理及执行顺序

## 一、核心架构

### 1. Spring Security 的三层防线

```
请求进入
  ↓
第一层：Filter Chain（过滤器链）
  ├─ SecurityContextPersistenceFilter - 恢复 SecurityContext
  ├─ TokenAuthenticationFilter - Token 认证（自定义）
  ├─ ExceptionTranslationFilter - 异常转换
  └─ FilterSecurityInterceptor - 权限检查
  ↓
第二层：Authentication（认证）
  ├─ 验证用户身份
  └─ 构建 Authentication 对象
  ↓
第三层：Authorization（授权）
  ├─ 方法级权限检查（@PreAuthorize）
  └─ URL 级权限检查
  ↓
响应返回
```

### 2. 核心组件关系图

```
HttpServletRequest
  ↓
FilterChain
  ├─ TokenAuthenticationFilter（提取 Token）
  │   ├─ 从 Header 获取用户（login-user）
  │   ├─ 从 Token 参数获取用户
  │   └─ 从 OAuth2 API 验证 Token
  │       ↓
  │   构建 LoginUser
  │       ↓
  │   SecurityFrameworkUtils.setLoginUser()
  │       ↓
  │   SecurityContextHolder.getContext().setAuthentication()
  │
  ├─ ExceptionTranslationFilter（异常处理）
  │   ├─ 捕获 AuthenticationException
  │   │   ↓ AuthenticationEntryPointImpl（返回 401）
  │   └─ 捕获 AccessDeniedException
  │       ↓ AccessDeniedHandlerImpl（返回 403）
  │
  └─ FilterSecurityInterceptor（权限检查）
      ├─ 检查 URL 权限
      └─ 检查方法权限
```

## 二、请求执行流程（详细步骤）

### 步骤 1：请求到达 Filter Chain

```
HTTP 请求 → Servlet 容器 → DispatcherServlet 前置 → Filter Chain
```

### 步骤 2：TokenAuthenticationFilter 执行

**位置**：`TokenAuthenticationFilter:47-80`

```java
protected void doFilterInternal(HttpServletRequest request,
                                HttpServletResponse response,
                                FilterChain chain) {
    // 步骤 2.1：尝试从 Header 获取用户（网关透传）
    LoginUser loginUser = buildLoginUserByHeader(request);

    // 步骤 2.2：如果 Header 中没有，尝试从 Token 获取
    if (loginUser == null) {
        String token = SecurityFrameworkUtils.obtainAuthorization(
            request,
            securityProperties.getTokenHeader(),      // "Authorization"
            securityProperties.getTokenParameter()    // "token"
        );

        if (StrUtil.isNotEmpty(token)) {
            Integer userType = WebFrameworkUtils.getLoginUserType(request);
            try {
                // 步骤 2.3：调用 OAuth2 API 验证 Token
                loginUser = buildLoginUserByToken(token, userType);

                // 步骤 2.4：如果验证失败，尝试 Mock 模式（开发调试）
                if (loginUser == null) {
                    loginUser = mockLoginUser(request, token, userType);
                }
            } catch (Throwable ex) {
                // 步骤 2.5：异常处理，返回错误响应
                CommonResult<?> result = globalExceptionHandler.allExceptionHandler(request, ex);
                ServletUtils.writeJSON(response, result);
                return;  // 中断过滤链
            }
        }
    }

    // 步骤 2.6：设置当前用户到 SecurityContext
    if (loginUser != null) {
        SecurityFrameworkUtils.setLoginUser(loginUser, request);
    }

    // 步骤 2.7：继续过滤链
    chain.doFilter(request, response);
}
```

### 步骤 3：Token 验证详解

**位置**：`TokenAuthenticationFilter:83-106`

```java
private LoginUser buildLoginUserByToken(String token, Integer userType) {
    try {
        // 步骤 3.1：调用 OAuth2 API 检查 Token
        OAuth2AccessTokenCheckRespDTO accessToken =
            oauth2TokenApi.checkAccessToken(token).getCheckedData();

        if (accessToken == null) {
            return null;  // Token 无效
        }

        // 步骤 3.2：验证用户类型是否匹配
        // 防止 admin-api 用户访问 app-api，或反之
        if (userType != null &&
            ObjectUtil.notEqual(accessToken.getUserType(), userType)) {
            throw new AccessDeniedException("错误的用户类型");
        }

        // 步骤 3.3：构建 LoginUser 对象
        return new LoginUser()
            .setId(accessToken.getUserId())
            .setUserType(accessToken.getUserType())
            .setInfo(accessToken.getUserInfo())
            .setTenantId(accessToken.getTenantId())
            .setScopes(accessToken.getScopes())
            .setExpiresTime(accessToken.getExpiresTime());

    } catch (ServiceException serviceException) {
        // Token 校验失败，返回 null（允许继续，由后续过滤器处理）
        return null;
    }
}
```

### 步骤 4：设置 SecurityContext

**位置**：`SecurityFrameworkUtils:125-151`

```java
public static void setLoginUser(LoginUser loginUser, HttpServletRequest request) {
    // 步骤 4.1：创建 Authentication 对象
    Authentication authentication = buildAuthentication(loginUser, request);

    // 步骤 4.2��设置到 SecurityContext（ThreadLocal）
    SecurityContextHolder.getContext().setAuthentication(authentication);

    // 步骤 4.3：额外设置到 Request 中（用于日志记录）
    if (request != null) {
        WebFrameworkUtils.setLoginUserId(request, loginUser.getId());
        WebFrameworkUtils.setLoginUserType(request, loginUser.getUserType());
    }
}

private static Authentication buildAuthentication(LoginUser loginUser,
                                                   HttpServletRequest request) {
    // 步骤 4.4：创建 UsernamePasswordAuthenticationToken
    UsernamePasswordAuthenticationToken authenticationToken =
        new UsernamePasswordAuthenticationToken(
            loginUser,              // principal（用户信息）
            null,                   // credentials（密码，Token 模式下为 null）
            Collections.emptyList() // authorities（权限列表）
        );

    // 步骤 4.5：设置请求详情
    authenticationToken.setDetails(
        new WebAuthenticationDetailsSource().buildDetails(request)
    );

    return authenticationToken;
}
```

### 步骤 5：ExceptionTranslationFilter 异常处理

**触发时机**：当后续过滤器抛出异常时

```
FilterSecurityInterceptor 抛出异常
  ↓
ExceptionTranslationFilter 捕获
  ├─ AuthenticationException（未认证）
  │   ↓
  │   AuthenticationEntryPointImpl.commence()
  │   ↓
  │   返回 401 Unauthorized
  │
  └─ AccessDeniedException（权限不足）
      ↓
      AccessDeniedHandlerImpl.handle()
      ↓
      返回 403 Forbidden
```

**位置**：`AuthenticationEntryPointImpl:36-40`

```java
@Override
public void commence(HttpServletRequest request,
                     HttpServletResponse response,
                     AuthenticationException e) {
    log.debug("[commence][访问 URL({}) 时，没有登录]", request.getRequestURI(), e);
    // 返回 401
    ServletUtils.writeJSON(response, CommonResult.error(UNAUTHORIZED));
}
```

**位置**：`AccessDeniedHandlerImpl:34-41`

```java
@Override
public void handle(HttpServletRequest request,
                   HttpServletResponse response,
                   AccessDeniedException e) throws IOException, ServletException {
    log.warn("[commence][访问 URL({}) 时，用户({}) 权限不够]",
             request.getRequestURI(),
             SecurityFrameworkUtils.getLoginUserId(), e);
    // 返回 403
    ServletUtils.writeJSON(response, CommonResult.error(FORBIDDEN));
}
```

## 三、完整请求时序图

```
时间轴 →

1. 请求到达
   ↓
2. TokenAuthenticationFilter
   ├─ 提取 Token
   ├─ 验证 Token
   └─ 设置 SecurityContext
   ↓
3. ExceptionTranslationFilter
   └─ 准备异常处理器
   ↓
4. FilterSecurityInterceptor
   ├─ 检查 URL 权限
   └─ 权限不足 → 抛出异常
   ↓
5. ExceptionTranslationFilter 捕获异常
   ├─ AuthenticationException → 401
   └─ AccessDeniedException → 403
   ↓
6. DispatcherServlet
   ├─ 路由到 Controller
   └─ 执行方法
   ↓
7. @PreAuthorize 检查方法权限
   ├─ 权限不足 → 403
   └─ 权限足够 → 执行方法
   ↓
8. 返回响应
```

## 四、关键概念解析

### 1. SecurityContext 与 ThreadLocal

```java
// SecurityContext 存储在 ThreadLocal 中
SecurityContextHolder.getContext()  // 获取当前线程的 SecurityContext

// 在同一线程中，任何地方都可以获取用户信息
LoginUser user = SecurityFrameworkUtils.getLoginUser();
Long userId = SecurityFrameworkUtils.getLoginUserId();
```

**注意**：异步线程中 ThreadLocal 会丢失，项目使用 `TransmittableThreadLocalSecurityContextHolderStrategy` 解决。

### 2. Authentication 对象结构

```java
Authentication {
    principal: LoginUser,           // 用户信息
    credentials: null,              // Token 模式下为 null
    authorities: [],                // 权限列表（空）
    authenticated: true,            // 是否已认证
    details: WebAuthenticationDetails // 请求详情
}
```

### 3. 三种获取用户的优先级

```
优先级 1：Header login-user（网关透传）
  ↓ 如果没有
优先级 2：Token 参数或 Header（OAuth2 验证）
  ↓ 如果没有
优先级 3：Mock 模式（开发调试）
  ↓ 如果都没有
用户为 null（允许继续，由权限检查决定是否拒绝）
```

### 4. 用户类型校验

```java
// 防止权限越界
if (userType != null &&
    ObjectUtil.notEqual(accessToken.getUserType(), userType)) {
    throw new AccessDeniedException("错误的用户类型");
}

// 场景：
// - /admin-api/* 要求 userType = ADMIN
// - /app-api/* 要求 userType = APP
// - /ws/* 不要求 userType
```

## 五、异常处理流程

### 场景 1：未登录访问受保护资源

```
请求 → TokenAuthenticationFilter
  ├─ 没有 Token
  └─ loginUser = null
  ↓
FilterSecurityInterceptor
  ├─ 检查权限
  └─ 抛出 AuthenticationException
  ↓
ExceptionTranslationFilter
  ├─ 捕获 AuthenticationException
  └─ 调用 AuthenticationEntryPointImpl.commence()
  ↓
返回 401 Unauthorized
```

### 场景 2：已登录但权限不足

```
请求 → TokenAuthenticationFilter
  ├─ 验证 Token 成功
  └─ loginUser 已设置
  ↓
FilterSecurityInterceptor
  ├─ 检查权限
  └─ 权限不足 → 抛出 AccessDeniedException
  ↓
ExceptionTranslationFilter
  ├─ 捕获 AccessDeniedException
  └─ 调用 AccessDeniedHandlerImpl.handle()
  ↓
返回 403 Forbidden
```

### 场景 3：Token 验证异常

```
请求 → TokenAuthenticationFilter
  ├─ 调用 OAuth2 API
  └─ 抛出 ServiceException
  ↓
catch 块捕获
  ├─ 记录日志
  └─ 返回错误响应
  ↓
中断过滤链，直接返回
```

## 六、配置与自定义

### 1. 安全配置

**位置**：`ZrkWebSecurityConfigurerAdapter:111-160`

```java
@Bean
protected SecurityFilterChain filterChain(HttpSecurity httpSecurity) throws Exception {
    httpSecurity
        // 启用 CORS
        .cors(Customizer.withDefaults())
        // 禁用 CSRF（无状态 API）
        .csrf(AbstractHttpConfigurer::disable)
        // 禁用 Session（使用 Token）
        .sessionManagement(session ->
            session.sessionCreationPolicy(SessionCreationPolicy.STATELESS))
        // 添加自定义 Token 过滤器
        .addFilterBefore(authenticationTokenFilter,
                        UsernamePasswordAuthenticationFilter.class)
        // 配置异常处理
        .exceptionHandling(exception ->
            exception
                .authenticationEntryPoint(authenticationEntryPoint)
                .accessDeniedHandler(accessDeniedHandler))
        // 配置权限规则
        .authorizeRequests(authorize -> {
            // 免登录 URL
            authorize.antMatchers(permitAllUrls).permitAll();
            // 其他 URL 需要认证
            authorize.anyRequest().authenticated();
        });

    return httpSecurity.build();
}
```

### 2. 权限配置属性

**位置**：`SecurityProperties:15-56`

```yaml
zrk:
  security:
    tokenHeader: "Authorization"      # Token 请求头
    tokenParameter: "token"           # Token 参数（WebSocket）
    mockEnable: false                 # Mock 模式开关
    mockSecret: "test"                # Mock 密钥
    permitAllUrls: []                 # 免登录 URL 列表
    passwordEncoderLength: 4          # 密码加密复杂度
```

## 八、常见问题

### Q1: 为什么需要三种获取用户的方式？

**A:**
- **Header login-user**：网关已认证，直接透传用户信息，性能最优
- **Token 验证**：直接调用 OAuth2 API，支持 Nginx 直转场景
- **Mock 模式**：开发调试，无需真实 OAuth2 服务

### Q2: 为什么要校验用户类型？

**A:** 防止权限越界。例如，admin 用户不应该访问 app-api，反之亦然。

### Q3: SecurityContext 为什么使用 ThreadLocal？

**A:** 每个请求在独立线程中处理，ThreadLocal 保证线程隔离，避免用户信息混乱。

### Q4: 异步任务中如何获取用户信息？

**A:** 使用 `TransmittableThreadLocal`，在异步任务提交前保存上下文，任务执行时恢复。

### Q5: 如何自定义权限检查？

**A:** 实现 `AuthorizeRequestsCustomizer` 接口，在 `ZrkWebSecurityConfigurerAdapter` 中注入。

## 总结

Spring Boot Security 的执行顺序：

1. **TokenAuthenticationFilter** - 提取并验证 Token，设置 SecurityContext
2. **ExceptionTranslationFilter** - 准备异常处理器
3. **FilterSecurityInterceptor** - 检查 URL 权限
4. **DispatcherServlet** - 路由到 Controller
5. **@PreAuthorize** - 检查方法权限
6. **异常处理** - 返回 401/403 错误

项目通过**无状态 Token 认证**、**多租户隔离**、**用户类型校验**等机制，实现了安全、高效的微服务认证授权体系。
