## 一、hashCode作用

#### 1. 提升哈希集合的查找效率

Java 中的哈希集合（HashMap、HashSet、HashTable 等）底层依赖哈希表实现，hashCode 的核心价值是

##### 快速定位对象在哈希表中的存储位置：

*   存储对象时：通过 hashCode 计算出对象应存放在哈希表数组的哪个索引位置，避免遍历整个数组。
*   查找 / 判断重复时：先通过 hashCode 定位到目标索引，再只对比该索引下的少量对象（通过 equals），而非对比所有对象。
    没有 `hashCode`，哈希集合的查找效率会从 O (1) 退化到 O (n)。

#### 2. 辅助 equals 方法

Object 类定义了 hashCode 和 equals 的核心约定：

*   若两个对象通过 equals() 比较相等，则它们的 hashCode() 必须返回相同值；
*   若两个对象的 hashCode() 返回不同值，则它们的 equals() 一定不相等；
*   （反之不强制）hashCode 相同的对象，equals 不一定相等（称为 “哈希冲突”）。

## 二、为什么要同时重写 equals 和 hashCode

#### 1. 先理解：默认的 equals 和 hashCode 行为

Object 类的默认实现：

*   equals()：直接比较两个对象的内存地址（只有同一个对象才返回 true）；
*   hashCode()：返回对象的内存地址转换后的整数（不同对象的 hashCode 几乎不同）

但实际开发中，我们通常需要按业务逻辑判断对象相等（比如：User 对象只要 id 相同就认为相等），这时候需要重写 equals—— 但如果只重写 equals 不重写 hashCode，会违反 Java 约定，导致哈希集合功能异常。

#### 2. 只重写 equals 的坑

假设我们定义一个 User 类，只重写 equals（按 id 判断相等），不重写 hashCode：

    class User {
        private Integer id;
        private String name;

        // 构造器、getter/setter 省略

        // 只重写 equals：id 相同则认为对象相等
        @Override
        public boolean equals(Object o) {
            if (this == o) return true;
            if (o == null || getClass() != o.getClass()) return false;
            User user = (User) o;
            return Objects.equals(id, user.id);
        }

        // 未重写 hashCode，使用 Object 类的默认实现（返回内存地址哈希值）
    }

    // 测试代码
    public class Test {
        public static void main(String[] args) {
            User u1 = new User(1, "张三");
            User u2 = new User(1, "张三");

            // 1. equals 判定相等
            System.out.println(u1.equals(u2)); // 输出 true

            // 2. 但 hashCode 不同（因为是两个不同对象，内存地址不同）
            System.out.println(u1.hashCode()); // 比如：123456
            System.out.println(u2.hashCode()); // 比如：789012

            // 3. 放入 HashSet（不允许重复），却被当成两个对象存储
            HashSet<User> set = new HashSet<>();
            set.add(u1);
            set.add(u2);
            System.out.println(set.size()); // 输出 2（预期应该是 1，因为 u1 和 u2 逻辑上相等）
        }
    }

**问题原因**：HashSet 存储时，先通过 hashCode 定位存储位置，u1 和 u2 的 hashCode 不同，会被存到哈希表的不同位置，即使 equals 相等，也会被当成两个不同对象。

#### 3. 正确做法：同时重写 equals 和 hashCode

    class User {
        private Integer id;
        private String name;

        // 构造器、getter/setter 省略

        // 重写 equals：按 id 判断相等
        @Override
        public boolean equals(Object o) {
            if (this == o) return true;
            if (o == null || getClass() != o.getClass()) return false;
            User user = (User) o;
            return Objects.equals(id, user.id);
        }

        // 重写 hashCode：基于 equals 中用到的字段（id）生成
        @Override
        public int hashCode() {
            // 使用 Objects.hash 工具方法，自动处理 null 值
            return Objects.hash(id);
        }
    }

    // 测试代码
    public class Test {
        public static void main(String[] args) {
            User u1 = new User(1, "张三");
            User u2 = new User(1, "张三");

            System.out.println(u1.equals(u2)); // true
            System.out.println(u1.hashCode() == u2.hashCode()); // true
            HashSet<User> set = new HashSet<>();
            set.add(u1);
            set.add(u2);
            System.out.println(set.size()); // 输出 1（符合预期，去重成功）
        }
    }

#### 4. 重写的核心原则

*   equals 中用到的字段（比如 id），必须全部参与 hashCode 的计算；
*   确保 “相等的对象必有相同的 hashCode”，不相等的对象尽量有不同的 hashCode（减少哈希冲突）；
*   推荐使用 Objects.hash(字段1, 字段2...) 生成 hashCode（自动处理 null，简洁且符合规范）。

## 哈希冲突

当两个不同的对象，通过 hashCode() 计算后（HashMap 还会对 hashCode 做二次哈希），最终得到了相同的哈希表数组索引位置，就发生了哈希冲突。

*   哈希冲突无法完全避免：因为 hashCode() 返回 int 类型（最多 2^32 个值），但程序中可能创建无限多的对象，必然会出现 “不同对象→相同 hashCode→相同索引” 的情况；
*   冲突的两种场景：
    1.  两个对象的 hashCode() 本身就相同；
    2.  hashCode() 不同，但经 HashMap 二次哈希 + 取模计算后，最终索引相同（更常见）。

HashMap 处理哈希冲突的核心方案是 “数组 + 链表 / 红黑树”（Java 8 及以后），双向链表和红黑树都是为了解决哈希冲突而设计的，目的是平衡冲突后的查询 / 插入效率。

## Objects.hash 核心原理

#### 1. 先看源码

    // Objects.hash 方法
    public static int hash(Object... values) {
        // 底层直接调用 Arrays.hashCode
        return Arrays.hashCode(values);
    }

    // Arrays.hashCode 方法（核心计算逻辑）
    public static int hashCode(Object a[]) {
        // 空数组返回 1（而非 0，避免和单个 null 的哈希值混淆）
        if (a == null)
            return 0;

        int result = 1;
        // 遍历所有参数，累加计算哈希值
        for (Object element : a) {
            // 处理 null：element 为 null 时贡献 0，否则取 element.hashCode()
            result = 31 * result + (element == null ? 0 : element.hashCode());
        }
        return result;
    }

#### 2. 原理拆解

Objects.hash(字段1, 字段2, ...) 的计算逻辑本质是加权累加哈希值，核心步骤：

*   步骤 1：将传入的多个字段封装成一个 Object\[] 数组；
*   步骤 2：初始化哈希结果 result = 1
*   步骤 3：遍历数组中的每个字段：
    *   若字段为 null，贡献的哈希值为 0；
    *   若字段非 null，贡献的哈希值为该字段自身的 hashCode()；
    *   执行 result = 31 \* result + 字段哈希值（核心加权逻辑）；
