## 一句话结论

Spring Bean 生命周期核心流程：**实例化 → 属性填充 → Aware 回调 → BeanPostProcessor 前置 → 初始化回调 → BeanPostProcessor 后置（AOP 代理在此生成）→ 使用 → 销毁回调**。

---

### 完整生命周期流程（面试必背⭐⭐⭐⭐⭐）

```
┌─────────────────────────────────────────────────────┐
│                    创建阶段                           │
├─────────────────────────────────────────────────────┤
│ 1. 实例化（Instantiation）                            │
│    - 反射调用构造函数创建对象（此时属性都是默认值）          │
│                                                     │
│ 2. 属性填充（Populate Properties）                    │
│    - @Autowired / @Value 注入依赖                     │
│    - 三级缓存解决循环依赖就发生在这一步                   │
│                                                     │
│ 3. Aware 接口回调                                     │
│    - BeanNameAware.setBeanName()                    │
│    - BeanFactoryAware.setBeanFactory()              │
│    - ApplicationContextAware.setApplicationContext() │
│                                                     │
│ 4. BeanPostProcessor.postProcessBeforeInitialization│
│    - 所有 BPP 的前置处理                               │
│    - @PostConstruct 在这里被处理                       │
│      (CommonAnnotationBeanPostProcessor)             │
│                                                     │
│ 5. 初始化回调（3 种方式，按顺序执行）                    │
│    ① @PostConstruct 方法                             │
│    ② InitializingBean.afterPropertiesSet()           │
│    ③ 自定义 init-method                               │
│                                                     │
│ 6. BeanPostProcessor.postProcessAfterInitialization  │
│    - ⭐ AOP 代理对象在这里生成                          │
│      (AbstractAutoProxyCreator)                      │
├─────────────────────────────────────────────────────┤
│                    使用阶段                           │
│    Bean 被应用代码使用                                 │
├─────────────────────────────────────────────────────┤
│                    销毁阶段                           │
│ 7. 销毁回调（3 种方式，按顺序执行）                      │
│    ① @PreDestroy 方法                                │
│    ② DisposableBean.destroy()                        │
│    ③ 自定义 destroy-method                            │
└─────────────────────────────────────────────────────┘
```

**记忆口诀**：实例化 → 填属性 → Aware → BPP 前 → PostConstruct → afterPropertiesSet → init-method → BPP 后（AOP）→ 用 → PreDestroy → destroy → destroy-method

### 关键环节详解

#### 1. BeanPostProcessor（BPP，最重要的扩展点）

```java
public interface BeanPostProcessor {
    // 初始化前调用（@PostConstruct 在此阶段被触发）
    default Object postProcessBeforeInitialization(Object bean, String beanName) {
        return bean;
    }
    // 初始化后调用（AOP 代理在此阶段生成）
    default Object postProcessAfterInitialization(Object bean, String beanName) {
        return bean; // 可以返回代理对象替换原始 Bean
    }
}
```

**Spring 内置的关键 BPP**：

| BPP | 职责 |
|-----|------|
| `AutowiredAnnotationBeanPostProcessor` | 处理 @Autowired / @Value 注入 |
| `CommonAnnotationBeanPostProcessor` | 处理 @PostConstruct / @PreDestroy / @Resource |
| `AbstractAutoProxyCreator` | **生成 AOP 代理对象** |
| `ApplicationContextAwareProcessor` | 处理各种 Aware 接口回调 |

#### 2. 三种初始化方式对比

| 方式 | 来源 | 执行顺序 | 推荐度 |
|------|------|---------|--------|
| `@PostConstruct` | JSR-250 标准 | **最先** | 推荐 |
| `InitializingBean.afterPropertiesSet()` | Spring 接口 | 第二 | 框架级 |
| `init-method`（XML 或 @Bean(initMethod)） | Spring 配置 | **最后** | 与 Spring 解耦 |

```java
@Component
public class MyBean implements InitializingBean {

    @PostConstruct  // ① 最先执行
    public void postConstruct() {
        System.out.println("1. @PostConstruct");
    }

    @Override  // ② 第二执行
    public void afterPropertiesSet() {
        System.out.println("2. afterPropertiesSet");
    }

    // ③ 最后执行（需要在 @Bean 中指定 initMethod）
    public void customInit() {
        System.out.println("3. custom init-method");
    }
}
```

#### 3. Aware 接口（获取 Spring 容器资源）

| 接口 | 注入的资源 | 使用场景 |
|------|----------|---------|
| BeanNameAware | 当前 Bean 的名称 | 日志、调试 |
| BeanFactoryAware | BeanFactory 引用 | 手动获取其他 Bean |
| ApplicationContextAware | ApplicationContext 引用 | 获取环境、事件发布等 |
| EnvironmentAware | Environment 引用 | 读取配置属性 |

> **推荐**：优先使用 @Autowired 注入 ApplicationContext，而非实现 Aware 接口（降低与 Spring 的耦合）

### Bean 作用域与生命周期的关系

| 作用域 | 创建时机 | 销毁时机 | 生命周期管理 |
|--------|---------|---------|------------|
| **singleton**（默认） | 容器启动时 | 容器关闭时 | Spring 完整管理 |
| **prototype** | 每次 getBean() 时 | **Spring 不管理销毁** | 创建后交给调用者 |
| request | 每个 HTTP 请求 | 请求结束 | Web 容器管理 |
| session | 每个会话 | 会话过期 | Web 容器管理 |

> **重要**：prototype 作用域的 Bean，Spring **不会调用销毁回调**（@PreDestroy 不会执行），需要调用者自行管理资源释放

### Bean 生命周期源码关键方法

```java
// AbstractAutowireCapableBeanFactory
protected Object doCreateBean(String beanName, RootBeanDefinition mbd, Object[] args) {
    // 1. 实例化
    BeanWrapper instanceWrapper = createBeanInstance(beanName, mbd, args);

    // 2. 属性填充（依赖注入）
    populateBean(beanName, mbd, instanceWrapper);

    // 3. 初始化（Aware + BPP + 初始化回调）
    Object exposedObject = initializeBean(beanName, exposedObject, mbd);

    return exposedObject;
}

protected Object initializeBean(String beanName, Object bean, RootBeanDefinition mbd) {
    // 3.1 Aware 回调
    invokeAwareMethods(beanName, bean);
    // 3.2 BPP 前置处理（@PostConstruct 在此触发）
    Object wrappedBean = applyBeanPostProcessorsBeforeInitialization(bean, beanName);
    // 3.3 初始化回调（afterPropertiesSet + init-method）
    invokeInitMethods(beanName, wrappedBean, mbd);
    // 3.4 BPP 后置处理（AOP 代理在此生成）
    wrappedBean = applyBeanPostProcessorsAfterInitialization(wrappedBean, beanName);
    return wrappedBean;
}
```

### 高频追问

1. **@PostConstruct 和 afterPropertiesSet 有什么区别？** @PostConstruct 是 JSR-250 标准注解，与 Spring 无耦合；afterPropertiesSet 是 Spring 接口方法。执行顺序：@PostConstruct 先
2. **AOP 代理是在 Bean 生命周期的哪个阶段生成的？** BPP 的后置处理阶段（`postProcessAfterInitialization`），由 `AbstractAutoProxyCreator` 生成
3. **为什么构造器注入能避免循环依赖的 NPE？** 构造器注入在实例化阶段就需要依赖，此时 Bean 还没进三级缓存，无法提前暴露 → 直接报错而非 NPE
4. **Spring 容器关闭时怎么触发销毁回调？** `context.close()` 或注册 ShutdownHook（`context.registerShutdownHook()`）
5. **prototype Bean 的 @PreDestroy 会执行吗？** 不会！Spring 不管理 prototype Bean 的完整生命周期，创建后就不再跟踪
